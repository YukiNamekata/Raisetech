# 第２回講座での学び

#### Git
* バージョン管理システムの 1 つで、今やほぼ主流。
* リモートリポジトリとローカルリポジトリ。分散型。
* コマンドgit clone git pull git add git commit git push
* ブランチを切って、変更を加えたあと「マージ（統合）」する。リモートリポジトリが加わると、ブランチをプッシュした後、プルリクエストを出すことになる。

#### Markdown
* HTML を習得しなくても綺麗なスタイルで文章が書ける。非常にシンプル。

#### GitHub TIPS
* Issue...プロジェクトにおける課題（改善、実装が必要なこと）、修正点を記録しておく場所。
* Files Changed...Pull Request で表示される、ファイルの変更数。
* 画像、図の挿入
